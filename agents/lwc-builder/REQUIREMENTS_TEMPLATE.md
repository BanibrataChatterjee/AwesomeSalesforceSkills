## 4. Security and accessibility posture

- All DOM event handlers live on semantic elements (`<button>`, `<a>`, `<input>`) — never `onclick` on a `<div>` or `<span>`.
- No raw `alert(...)` or `document.*` calls; surface messages via `LightningAlert`, `LightningConfirm`, or `ShowToastEvent`.
- Every data read goes through a `@wire` adapter (`getRecord`, `getListUi`, `@salesforce/apex/...`) — the scratch compiler enforces that these imports resolve.
- When `emit_controller=yes`, the Apex controller routes every DML through `templates/apex/SecurityUtils.cls` and asserts CRUD + FLS explicitly. `@AuraEnabled(cacheable=true)` is only used on pure-read methods.
- Accessibility target is `{{a11y_tier}}`. Decorative images carry `alt=""`; interactive images carry a meaningful `alt`. Focus order matches visual order.

## 5. Test plan

- **Library-only Jest:** `__tests__/{{component_name}}.test.js` renders the component with a minimal wire mock and asserts the primary DOM shape (one user-visible element + one branch).
- **Wire edge cases:** error state via `emitError` on the adapter, empty state via `emitData(null)`.
- **Accessibility smoke:** every emitted interactive element has an accessible name (label, aria-label, or text content).
- **Coverage target:** ≥ 80% lines on the component JS.

Gate C does NOT run Jest — the scratch org can't. Local devs run `npm test` against the emitted bundle during development.
