# Official Source Selection

## Policy

Official Salesforce docs are the primary authority for:

- product behavior
- API semantics
- metadata behavior
- security requirements

Salesforce Architects content is the primary authority for:

- architecture patterns
- anti-patterns
- Well-Architected framing

## Practical Rule

Use local knowledge to sharpen guidance, not to override official behavior claims unless the skill explicitly documents a platform nuance or gap.
